Fine-grained lifetime management

I quite like the Mutex API in Rust, since it's pretty easy to manage critical sections by deciding when to drop a MutexGuard. However, it's a little too easy to accidentally deadlock by forgetting when to drop a MutexGuard. 

let mutex = std::sync::Mutex::new(());
let guard1 = mutex.lock().unwrap();
// do some other stuff...
let guard2 = mutex.lock().unwrap();
// oops! deadlocked!

The example above is a little silly, in that it's pretty easy to catch this one. More practically speaking, it's common to accidentally forget to drop a MutexGuard at the exact instant it's no longer leading, which can cause critical sections of code to be way longer than needed. This results in mysterious performance losses
